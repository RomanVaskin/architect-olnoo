# Reviewer Agent

**Version:** 0.2.0
**Status:** Draft — Phase 4 MVP implemented
**Last Updated:** 2026-07-20

> Спецификация агента-ревьюера, проверяющего результаты работы других агентов пайплайна.

---

## Назначение документа

Описывает роль Reviewer Agent в AI-пайплайне: какие данные он принимает на вход, критерии проверки и что возвращает.

## Реализация Phase 4

После успешной генерации каждого варианта отдельный Reviewer сравнивает два изображения: подтверждённый Primary View и готовую концепцию. В MVP используется стабильная vision-модель Gemini `gemini-2.5-flash` (серверная переменная `GEMINI_REVIEW_MODEL` позволяет заменить её без изменения контракта). Провайдер скрыт за `GeometryReviewProvider` и `getGeometryReviewer`, поэтому API, хранилище и интерфейс не зависят от Gemini.

Пользователь может отключить проверку перед платным запуском. Интерфейс заранее сообщает, что для каждого созданного варианта выполняется отдельный AI-запрос, который может увеличить стоимость и время обработки.

### Проверяемые признаки

- ракурс и перспектива;
- основные объёмы и этажность;
- геометрия крыши;
- положение оконных и дверных проёмов;
- общие пропорции и пятно здания.

Изменения материалов, цветов, освещения, ландшафта и погоды не считаются геометрическими расхождениями.

### Результат

Reviewer возвращает структурированный отчёт с одним из статусов:

- `no-obvious-deviations` — явных расхождений не обнаружено по всем пяти признакам с достаточной уверенностью;
- `possible-deviations` — обнаружено хотя бы одно вероятное расхождение с высокой уверенностью;
- `inconclusive` — изображений недостаточно для уверенного вывода;
- `not-run` — проверка отключена или не завершилась.

Каждый признак содержит статус, оценку уверенности 0–1 и короткое объяснение. Ответ модели повторно валидируется кодом; неполный, повторяющийся или некорректный набор признаков отклоняется.

### Гарантии честности и отказоустойчивость

- Отчёт является только предварительным визуальным AI-анализом, а не профессиональным заключением, обмером, сертификацией или проверкой строительных норм.
- В интерфейсе никогда не используется утверждение «геометрия проверена». Даже при отсутствии явных расхождений всегда показывается «Требует проверки специалиста».
- Ошибка Reviewer не превращает успешно созданную концепцию в ошибку и не удаляет уже оплаченный результат: изображение сохраняется с отчётом `not-run`.
- В логи не попадают изображения, промпты и сырые сообщения провайдера — только безопасный тип ошибки.
- Повторное локальное сохранение отчёта и изображения не вызывает новый AI-запрос.
